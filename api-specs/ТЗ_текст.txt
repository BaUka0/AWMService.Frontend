Техническое заданиена разработку микросервиса "Учет и контроль курсовых, дипломных и диссертационных работ"1. Цель проектаСоздание микросервиса, предназначенного для автоматизации процессов:подачи и утверждения тем курсовых, дипломных работ и диссертаций;назначения научных руководителей;отслеживания статуса выполнения работы;прикрепления текстов работ и отчетов по ним;фиксации результатов предзащиты и защиты;прикрепление подтверждающей справки антиплагиата;хранения и поиска работ;аналитики и отчетности по статусам, тематикам, оценкам и руководителям.Микросервис должен учитывать интеграцию с AD для авторизации и регистрацию/авторизацию внутри микросервиса. 2. Основные типы работКурсовая работа / проектДипломная работа / бакалавриатМагистерская диссертацияДокторская диссертация 3. Основной функционал микросервиса3.1. Жизненный цикл работы (workflow):Выбор темы студентом (или предложение собственной)Подача заявки на утверждение темы через личный кабинетУтверждение темы научным руководителемНазначение даты предзащитыПрикрепление черновика работы для предзащитыФиксация оценки за предзащитуДоработка работы (при необходимости)Подача на основную защитуЗагрузка окончательного варианта работыФиксация оценки за защитуФормирование финального отчета по работе4. Пользовательские роли и доступыРольОсновные функцииСтудентВыбор/предложение темы, прикрепление файлов, отслеживание статуса, просмотр комментариев и оценок. Выбор научного руководителя. Научный руководительУтверждение тем, комментирование, оценивание, рекомендация к защите. Список закрепленных обучающихся.Заведующий кафедройКонтроль выполнения работ по кафедре, статистика, подтверждение науч. Руководителей ПОДПИСЬ, ДатаСекретарь/менеджер ОРКонтроль сроков, формирование списков на защиту, печать отчетовЧлены комиссии / экспертПросмотр работ, выставление оценок за защитуПроректор по УРДоступ к сводной статистике по всем видам работ, институтам и кафедрамСистемный администраторУправление ролями, настройками, интеграцией, логами5. Интеграция с образовательным порталом SSOПолучение:ФИО, ИИН, ID студентаОбразовательная программа, курс, кафедра, институтСписок преподавателейУспеваемость (для допуска к защите)Синхронизация:Автоматическое обновление списков студентов по дисциплине "Дипломный проект" и "Курсовая работа"6. Поиск и фильтрация работФильтрация по параметрам:Тип работы (курсовая, дипломная, диссертация)Статус (на утверждении, на предзащите, допущен к защите, завершена и т.д.)Год, семестрКафедра, институтНаучный руководительФИО студента, ИИНОценка (предзащита, защита)7. Статистика и отчёты7.1. ДешбордыКол-во работ по статусам (предзащита, защита, завершено)Кол-во работ по кафедрам, институтамСводка по научным руководителям (нагрузка)Диаграммы по итоговым оценкам7.2. ВыгрузкиСписки студентов, допущенных к защитеВедомости по предзащите / защитеPDF- или Excel-отчеты по выборкеРеестр тем и прикреплённых работ8. Дополнительные функцииКомментарии студенту от научного руководителяИстория изменений по теме, загрузкам, статусамНапоминания и уведомления (о сроках, назначении даты защиты, необходимости загрузки)Фиксация отказов и смены темАрхивация завершенных работПоиск по ключевым словам в теме / аннотации работыАнтиплагиат-интеграция (если предусмотрено)9. Технические требованияАрхитектура: МикросервиснаяFrontend: ReactBackend: .Net, C#БД: MSSQLИнтеграции: SSO API (OAuth 2.0 / JWT)Форматы документов: PDF, DOCX, ZIP (архив работы + доп. материалы)Язык интерфейса: Русский, казахский (в перспективе мультиязычность)10. БезопасностьАутентификация через SSOРолевое разграничение прав доступаШифрование хранимых данныхЛогирование всех операцийВозможность восстановления истории11. Этапы реализацииСбор и уточнение требованийПроектирование архитектуры и БДРазработка модулей:Личный кабинет студентаКабинет научного руководителяИнтерфейсы кафедры, ОР, проректораИнтеграция с SSOТестированиеВнедрение в эксплуатациюПоддержка и доработка
